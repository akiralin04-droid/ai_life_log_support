# 🎙️ AI Life Log Support

**サービスURL**:[https://ailifelog2026.com](https://ailifelog2026.com)  
（※ゲストログイン機能を用意しておりますので、ワンクリックで全機能をお試しいただけます）

## サイト概要
### サイトテーマ
**「日々の記録（日記）」から「企業の資産（詳細なフィードバック）」をAIが創り出す**、次世代のライフログ＆マーケティングプラットフォームです。

### 扱うコンテンツの定義：体験型コンテンツ（Experience Content）
本アプリでは、単なる商品スペックではなく、ユーザーの**「体験」**に重きを置いた以下のコンテンツをレビュー対象とします。
*   **時間消費型体験**：映画、本、ゲーム、舞台など（没入した時間に対する感想）
*   **五感体感型体験**：飲食、ガジェット、旅行先、サウナなど（身体的・感覚的な感想）

### テーマを選んだ理由
私は読書や商品購入後の「感想」を残したいと思う一方で、文章を整理する手間や、「何を書けばいいか分からない」という悩みを感じていました。
一方で、商品や作品を提供する**企業側**も、単なる評価点数だけではない、**「具体的な改善点」や「ターゲット層のリアルな反応」**を求めていますが、従来のアンケート形式では回答率が低いという課題があります。

そこで、**「AIとの対話」**と**「キャンペーン（インセンティブ）」**を組み合わせることで、課題を解決できると考えました。
ユーザーはキャンペーンに参加し、AIからのインタビューに答える形式で日記を書くだけ。AIは企業が知りたいポイントを自然に聞き出し、質の高いレビューを生成します。
「ユーザーの負担軽減」と「企業のマーケティング収集」を両立する仕組みを実現したく、このテーマを選びました。

## 💡 開発の変遷（当初の想定と、実際のアップデート）
本プロジェクトはアジャイル開発の手法を取り入れ、MVP（最小限のプロダクト）からスタートしました。開発を進める中でUX（ユーザー体験）を見直し、当初の想定から以下のような強力なアップデートを行って完成させています。

1. **入力方式の進化（タイピングから音声認識へ）**
   * 【当初の想定】テキストベースでAIとチャットを行い、レビューを生成する。
   * 【実際の改善】テキスト入力すら手間に感じるユーザーの課題を解決するため、**ブラウザ標準の Web Speech API による音声入力**を導入。「マイクに向かって雑多な独り言を話すだけ」でAIが文脈を補完し、綺麗な日記やレビューへ推敲する「摩擦レス」な入力体験を実現しました。
2. **「無限推敲ループ」と「感情スコアギャップ分析」の追加**
   * 【当初の想定】AIが一度生成したものをユーザーが直接編集する。
   * 【実際の改善】「〇〇じゃなくて〜」というメタ発言もAIが汲み取り、何度でも対話で直せる「無限推敲ループ」を実装。さらに、ユーザーの「自己評価」とAIによる「客観的な感情スコア」の差分（ギャップ）を分析する機能を管理者ダッシュボードに追加し、ユーザーの無意識の心理的インサイトを可視化できるようにしました。

## 💻 実装済みの機能一覧

### 🤖 AI機能（コア機能）
* **AI音声チャット日記**：音声で雑多に話しかけるだけで、AIが整った日記文章へ変換・推敲。
* **企業連携型インタビュー**：キャンペーン対象商品の場合、企業が聞きたいポイントをAIが自然に質問。
* **AIレビュー自動生成**：対話の履歴（生データ）を元に、読みやすく整理された公開用レビュー記事を生成。

### 👤 ユーザー機能
* **日記・レビュー管理**：日記の公開/非公開設定、生成されたレビューの編集・投稿。
* **キャンペーン参加**：開催中のキャンペーンに参加し、特定のテーマでレビューを作成。
* **マイページ**：月間カレンダー（感情スコアによる色分け）、週間の未分析日記を集めた「ウィークリーAI振り返りレポート」の自動生成。
* **コミュニケーション**：他ユーザーの公開レビューへの「いいね」「コメント」、検索機能。
* **ゲストログイン機能**：ワンクリックで閲覧用アカウントとしてログイン可能。

### 🛡️ 管理者ダッシュボード機能
* システム全体のサマリー（ユーザー、キャンペーン、レビュー稼働状況）表示。
* **感情スコアのギャップ分析パネル**（ユーザー自己評価 vs AI評価）。
* ユーザーとAIの対話履歴（推敲前の音声生データ）のモニタリング機能。
* 不適切コンテンツの非公開・完全削除、ユーザーの権限管理。

## 🛠 開発環境と、実運用を見据えた技術的こだわり

* **バックエンド**: Ruby 3.3.6 / Ruby on Rails 8.0.4
* **データベース**: MySQL 8.0
* **フロントエンド**: HTML / CSS / JavaScript / Bootstrap 5 / Hotwire (Turbo) / Web Speech API
* **インフラ**: AWS (EC2 / Route53) / Nginx / Puma / SSL化済み
* **外部API**: OpenAI API (gpt-3.5-turbo)

**🔥 プロの現場を想定したこだわりの実装**
1. **Rails 8 標準機能によるPWA（Progressive Web App）対応**: マニフェストファイルとServiceWorkerを設定し、スマホの「ホーム画面に追加」からネイティブアプリのように全画面で起動可能にしました。
2. **パフォーマンスチューニング（N+1問題の徹底排除）**: `includes` を最適に配置し、一覧画面やカレンダー表示時の無駄なSQLクエリを一掃しました。
3. **強固なフェイルセーフ**: OpenAI APIのタイムアウト時でもアプリをクラッシュさせず、ユーザーが入力した「音声生データ」だけは絶対にロストさせずに引き継ぐエラーハンドリングを実装しました。
4. **RSpecによる自動テスト**: 権限のないユーザーが管理者画面へアクセスできないか等、FactoryBotを用いたセキュリティ・アクセス制御のテストを実装しています。

## ターゲットユーザと主な利用シーン
* **【一般ユーザー】**：帰り道にスマホのマイクに向かって「今日見た映画、ラストが衝撃的でさ…」と雑多に話しかける。AIがそれを整った日記文章に変換し、さらに「映画のレビューも共有しませんか？」と提案し、記事を自動生成する。
* **【企業・提供者】**：自社商品のプロモーションキャンペーンを開催し、点数だけでなく、AIが引き出した「顧客のリアルな感情や文脈を含んだフィードバックデータ」を収集する。
